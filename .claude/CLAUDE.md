Never cast a type `as any`
Only 1 React component per file, typically <100 LOCs
Tailwind color classes (e.g. `text-slate-700`, `bg-sky-500`) are defined in paletteV3, NOT in the default Tailwind palette. Always check paletteV3 for available colors — never use inline `style={{color: ...}}` when a Tailwind class exists
Always write new mutations using the `useMutation` hook pattern (see `useShareTopicMutation.ts` or `useAddPokerTemplateMutation.ts`): a `use*Mutation` hook that wraps `useMutation`, returns `[execute, submitting] as const`, and merges caller config via `...config`. Put default `updater`/`optimisticUpdater` before the `...config` spread so callers can still override them. This replaces the older `commitMutation` + `StandardMutation` + `useMutationProps` pattern — components get `submitting` from the hook, call the hook at the top level, and pass `onCompleted`/`onError`/`variables` directly in the execute config. When a mutation's updater is also reused by a subscription (e.g. `TeamSubscription`), export it as a named `SharedUpdater` from the same hook file. Do NOT create new files using the old `StandardMutation` pattern.
